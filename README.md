# 项目计划

## 角色

- 你是一个顶尖的UI/UE专家

## 项目重新构思与设计

### 项目概述

这是一个 Whistle 的插件，主要用于功能级别的接口代理和 response 的 mock 功能，配备美观的用户界面。

### 核心功能

#### 1. 功能管理
- **功能集合**：按业务功能或场景组织接口代理规则
- **状态控制**：一键启用/禁用整个功能集合
- **批量操作**：导入/导出功能配置，便于团队共享

#### 2. 接口代理方式
- **URL 重定向**：将请求转发到指定的其他 URL
- **本地文件代理**：返回本地文件内容
- **数据模板**：直接返回自定义的数据结构
- **动态数据生成**：支持 Mock.js 语法生成动态数据

#### 3. 高级功能
- **请求参数匹配**：根据请求参数不同返回不同响应
- **延时模拟**：模拟网络延迟
- **状态码模拟**：自定义 HTTP 状态码
- **历史记录**：记录代理历史，方便回溯

### 界面设计

#### 1. 主界面
- 左侧：功能列表导航栏
- 右侧：功能详情及接口配置区域
- 顶部：全局搜索和工具栏

#### 2. 功能管理界面
- 功能列表视图（支持列表/卡片切换）
- 每个功能卡片显示：
  - 功能名称
  - 状态标识（启用/禁用）
  - 包含的接口数量
  - 快捷操作按钮

#### 3. 接口配置界面
- 接口列表
- 接口详情编辑区：
  - 基本信息（URL、方法等）
  - 代理类型选择器
  - 响应数据编辑器（支持 JSON、XML 等格式）
  - 规则条件配置

#### 4. 响应编辑器
- 支持语法高亮
- JSON 格式化
- 模板变量支持
- 预览功能

### 技术实现

#### 前端技术栈
- React 构建 UI
- Ant Design 或 Material-UI 提供组件库
- Monaco Editor 用于代码编辑
- 状态管理：Redux 或 React Context

#### 后端实现
- 基于 Whistle 插件 API
- 数据存储：本地文件系统或 IndexedDB
- 规则解析引擎

### 用户流程

1. 创建新功能
2. 在功能下添加接口代理规则
3. 配置接口代理方式和响应数据
4. 启用功能进行测试
5. 根据需要调整配置
