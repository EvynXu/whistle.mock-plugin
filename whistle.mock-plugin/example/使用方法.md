# 413 Payload Too Large 错误模拟和问题解决

## 问题解决

1. 已将服务器请求体大小限制从 3MB 增加到 50MB，可处理更大的请求数据。
2. 添加了一个模拟 413 错误的接口配置，用于测试客户端错误处理逻辑。

## 使用方法

### 测试 413 错误响应

访问以下接口可以测试 413 错误响应：
```
http://your-api-domain/api/payload-too-large
```

此接口会返回 HTTP 状态码 413 和以下内容：
```json
{
  "code": 413,
  "message": "请求体太大，超出服务器限制",
  "error": "Payload Too Large"
}
```

### 在 Whistle 中添加规则

在 Whistle 中添加如下规则：
```
pattern whistle.mock-plugin://
```

如果想针对特定接口，可以配置：
```
example.com/api/upload whistle.mock-plugin:///api/payload-too-large
```

## 如何将配置添加到您的数据中

1. 在 Whistle 中访问 mock-plugin 的管理页面
2. 创建一个新的"错误响应模拟"功能模块（如果不存在）
3. 在该模块下添加名为"413 Payload Too Large 错误"的接口配置
4. 按照 example/interface-413.json 中的配置填写表单
