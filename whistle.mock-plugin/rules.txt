# Whistle Mock Plugin 使用指南
# 版本: 0.3.0

# ===== 基本规则配置 =====
# 在 Whistle 的规则配置中，使用以下格式添加规则：
* whistle.mock-plugins://

# 该插件现在会自动匹配已配置的接口规则，对于没有在插件中配置的接口，不会进行拦截

# ===== 规则匹配逻辑 =====
# 1. 请求会先检查是否匹配接口配置中定义的 URL 模式
# 2. 如果找到匹配的接口配置，且该接口处于启用状态，才会由插件处理
# 3. 如果没有匹配的接口配置，请求会直接透传到真实服务器

# ===== 常见配置示例 =====
# 不需要添加其他规则，只需要保留上面的全局规则即可
# 插件会根据在插件界面中配置的接口规则自动决定哪些请求需要处理

# ===== 调试功能 =====
# 通过访问以下URL可以查看插件的配置和状态：
# 查看全部配置: whistle.mock-plugin/_debug/config
# 查看启用的接口: whistle.mock-plugin/_debug/enabled-interfaces
